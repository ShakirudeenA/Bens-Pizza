Data Cleaning Process

The dataset contained several deliberate issues that needed to be addressed:

  • NULL values - Identified and handled inappropriate NULL values in various fields
  • Inconsistent categories - Standardized item categories across the dataset
  • Price anomalies - Corrected outlier prices (e.g., items priced at $1500 or $0)
  • Missing timestamps - Added missing created_at values based on similar orders
  • City name inconsistencies - Standardized city names (e.g., "Philly" vs "Philadelphia")
  • Order total miscalculations - Recalculated and corrected order totals

After cleaning, the data was normalized to 3NF (Third Normal Form) by creating separate tables for customers, menu items, addresses, orders, and order items.

Reflections & Learnings

It was an interesting experience to create a dashboard with a business's goals in mind. It made me factor in relevancy and significance, as it wasn't just about making aesthetically pleasing visuals, but creating ones that made sense in context.

Though I generated the dataset, in hindsight it was more limited than I expected. This highlighted the importance of a well-rounded dataset, since the more data you collect, the broader and more valuable the insights you can deliver.

The project reinforced the importance of data quality and proper normalization before visualization, as clean data is essential for accurate insights.
